# PLAN.md — ТЗ проекта ProfBot

## 1) Общее

**Проект:** Telegram-бот ProfBot для фриланс/IT-сообщества с внутренней валютой и магазином.  
**Исполнитель:** Павел.  
**Версия документа:** 1.0.  
**Дата:** актуализируется при каждом обновлении.

### 1.1 Назначение
Пользователи получают внутреннюю валюту («кристаллы») за активность и тратят её в магазине (доступы, консультации, промокоды).

### 1.2 Аудитория
- Основная: программисты 18–35 лет.
- Администраторы: 2–3 человека.

### 1.3 Платформа
- Telegram Bot API.
- Работа в мобильном и десктоп-клиенте Telegram.

---

## 2) Функциональные требования

### 2.1 Регистрация (`/start`)

**Сценарий:**
1. Пользователь отправляет `/start`.
2. Бот регистрирует пользователя по `telegram_id`.
3. В БД создаётся запись пользователя.
4. Начисляется приветственный бонус: **100 кристаллов**.

**Поля пользователя:**
- `telegram_id` (PK)
- `username`
- `first_name`
- `last_name` (nullable)
- `registered_at`
- `balance` (default: 100)
- `referral_code` (unique)
- `invited_by` (nullable)

### 2.2 Профиль (`/profile`)

**Вывод:**
- Основные данные: ID, имя, username, дата регистрации, баланс.
- Статистика: приглашённые, заработано по рефералке, потрачено, число транзакций.
- Реферальная ссылка.
- Последние 5 транзакций.

### 2.3 Реферальная система (`/referral`)

- Генерация уникального кода из 6 символов.
- Ссылка вида: `https://t.me/profbot?start={code}`.
- Бонус начисляется **один раз** за уникального приглашённого.
- Бонус: +50 пригласившему и +50 новому пользователю.

### 2.4 Внутренняя валюта — «кристаллы»

#### Начисления
| Действие | Кристаллы | Примечание |
|---|---:|---|
| Регистрация | 100 | Новому пользователю |
| Приглашение друга | 50 | Обоим участникам |
| Ежедневный бонус | 5–30 | По прогрессии |
| Админ-начисление | любое | Через админ-команду |

#### Списания
| Действие | Кристаллы | Примечание |
|---|---:|---|
| Покупка товара | цена товара | При подтверждении |
| Админ-списание | любое | Через админ-команду |

#### Логирование
Все операции по балансу хранятся в таблице `transactions`.

### 2.5 Ежедневный бонус (`/daily`)

- Получение бонуса: 1 раз в 24 часа.
- Прогрессия по серии дней: `5 → 7 → 10 → 15 → 20 → 25 → 30`.
- При пропуске серии — сброс на 5.

### 2.6 Магазин (`/shop`)

**Модель товара:**
- `name`, `description`, `price`
- `delivery_type` (`auto` / `manual`)
- `delivery_content`
- `is_active`

**Сценарии:**
- `auto`: списание + автоматическая выдача контента.
- `manual`: списание + создание заявки + уведомление пользователя.

---

## 3) MVP (минимальный срез)

В MVP должны войти:
1. `/start` (регистрация + 100 кристаллов).
2. Базовый профиль (`/profile`).
3. Реферальный код и базовая логика начисления.
4. Учёт баланса и журнал транзакций.
5. Минимальный каталог магазина (1–3 товара).
