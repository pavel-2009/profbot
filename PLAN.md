# Техническое задание — ProfBot

> 💡 Телеграм-бот ProfBot — платформа для фриланс‑сообщества с внутренней валютой и магазином.

- Заказчик: [Имя]
- Исполнитель: Павел
- Версия: 1.0
- Дата: 2026-04-30

## 1. Общие сведения
### 1.1. Назначение
Бот для монетизации сообщества программистов.
Пользователи получают внутреннюю валюту (кристаллы) за действия (приглашения друзей, ежедневные заходы и т. п.) и тратят кристаллы на товары (доступ к закрытым чатам, консультации, промокоды).

### 1.2. Аудитория
- Основная: программисты 18–35 лет
- Администраторы: 2–3 человека

### 1.3. Платформа
- Telegram Bot API
- Доступ через любой клиент Telegram (мобильный/десктоп)

## 2. Требования к функционалу
Включает: автоматическую регистрацию (/start), профиль (/profile), реферальную систему (/referral), кристаллы и логирование транзакций, ежедневный бонус (/daily), магазин (/shop, auto/manual delivery), пополнение (ЮKassa), интеграции API (/weather, /meme), статистику и графики (/mystats, график баланса, /rating), админ‑панель (/admin), клавиатуры и навигацию.

### 2.1 Регистрация
Поля пользователя: telegram_id, username, first_name, last_name, registered_at, balance=100, referral_code, invited_by.
Приветствие:
- 👋 Добро пожаловать, {first_name}
- ID в системе: {telegram_id}
- 🎁 +100 кристаллов
- /help для списка команд

### 2.2 Профиль
Вывод: ID, имя, username, дата регистрации, баланс, статистика, реферальная ссылка, последние 5 транзакций.
Кнопки: полная история, поделиться рефералкой, график.

### 2.3 Рефералка
- Уникальный код 6 символов (буквы+цифры)
- Ссылка `https://t.me/profbot?start={code}`
- Привязка invited_by, бонус один раз на уникального пользователя
- Начисление обоим +50 и лог транзакций

### 2.4 Экономика
- Начисления: регистрация 100, приглашение 50/50, daily 5–30, админ-начисления
- Списания: покупка, админ-списания
- Таблица transactions обязательна

### 2.5 Daily
- Раз в 24 часа
- Прогрессия: 5/7/10/15/20/25/30+
- Пропуск дня сбрасывает streak

### 2.6 Магазин
- Таблица products
- `delivery_type`: auto/manual
- Покупка: проверка баланса, списание, лог, выдача/создание manual-заказа и уведомление админов
- Таблица pending_orders (orders)

### 2.7 Платежи ЮKassa
- Тестовый и боевой режимы
- Пакеты пополнений и произвольная сумма
- payment_url, webhook, статус succeeded, начисление, лог
- Таблица payments

### 2.8 Внешние API
- /weather {город} (OpenWeatherMap)
- /meme {top} {bottom} (Imgflip)
- Альтернативы: CoinGecko, Cat Facts

### 2.9 Статистика и графики
- /mystats: активность, экономика, рефералы (включая активных и конверсию), магазин
- График баланса за 30 дней (matplotlib)
- /rating: топ-10 + место пользователя

### 2.10 Админ-панель
Разделы: пользователи, балансы, товары, общая статистика, настройки, manual-заказы.

### 2.11 Навигация
Главное меню: Главная, Магазин, Профиль, Пополнить, Рейтинг, Статистика, Рефералка, Бонус, Помощь.
Inline: пагинация, назад, в главное меню.

## 3. Нефункциональные требования
- Производительность: <1s (без внешних API), async stack, 1000 concurrent
- Безопасность: секреты в .env, проверка админов
- Надёжность: graceful shutdown, reconnect DB/Redis, error logging
- Масштабируемость: PostgreSQL, Redis, Docker

## 4. База данных
PostgreSQL: users, transactions, products, payments, orders, referrers (опционально).
Redis: FSM, кэш рейтинга, кэш топ-товаров, кэш курса кристалла.

## 5. Требования к коду
- Структура проекта как в ТЗ
- mypy strict, flake8 clean, black, isort
- Логирование INFO/WARNING/ERROR, действий и traceback, ротация
