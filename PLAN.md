1. Общие сведения
1.1. Назначение
Бот для монетизации сообщества программистов.
Пользователи:
получают внутреннюю валюту (кристаллы) за действия (приглашения друзей, ежедневные заходы и т. п.)
тратят кристаллы на товары (доступ к закрытым чатам, консультации, промокоды)
1.2. Аудитория
Основная: программисты 18–35 лет
Администраторы: 2–3 человека
1.3. Платформа
Telegram Bot API
Доступ через любой клиент Telegram (мобильный/десктоп)

2. Требования к функционалу
2.1. Автоматическая регистрация
Команда: /start
Сценарий:
Пользователь → /start
Бот → приветственное сообщение, регистрация по Telegram ID
Бот → создаёт запись в БД с полями:
  - telegram_id (integer, primary key)
  - username (string, из Telegram)
  - first_name (string)
  - last_name (string, может быть пустым)
  - registered_at (datetime)
  - balance (integer, default=100 бонусных кристаллов)
  - referral_code (string, уникальный)
  - invited_by (integer, telegram_id пригласившего, может быть NULL)
Текст приветствия:
👋 Добро пожаловать, {first_name}!

Ты автоматически зарегистрирован в системе "ProfBot".
Твой ID в системе: {telegram_id}

🎁 Тебе начислено 100 кристаллов за регистрацию!

Используй /help, чтобы увидеть список команд.

2.3. Реферальная система
Принцип работы
Для каждого пользователя генерируется уникальный реферальный код (6 символов: буквы+цифры)
Ссылка: https://t.me/profbot?start={code}
При переходе по ссылке новый пользователь привязывается к пригласившему
Бонус начисляется только один раз за уникального пользователя (по telegram_id)
При повторном переходе по ссылке того же пользователя бонус не начисляется
Сценарий приглашения
Пользователь A → /referral
Бот → показывает ссылку и текст:

🔗 ВАША РЕФЕРАЛЬНАЯ ССЫЛКА:
<https://t.me/profbot?start=XYZ789>

За каждого приглашённого друга вы получите 🔮 50 кристаллов.
Друг также получит 🔮 50 кристаллов при регистрации.

Приглашено: 3 человека
Заработано: 150 кристаллов

Список приглашённых:
✅ @ivan (14.04.2026)
✅ @petr (15.04.2026)
✅ @maria (16.04.2026)
Сценарий перехода по ссылке
Пользователь B → переходит по ссылке @username?start=XYZ789
Бот → проверяет код в start-параметре
Бот → находит пользователя A по коду
Бот → создаёт пользователя B с invited_by = telegram_id(A)
Бот → начисляет A: +50 кристаллов
Бот → начисляет B: +50 кристаллов
Бот → логирует транзакции у обоих:
  "Бонус за приглашение пользователя @username"

2.5. Ежедневный бонус
Команда: /daily или кнопка «🎁 Ежедневный бонус»
Механика:
бонус раз в 24 часа
при серии дней подряд сумма растёт
при пропуске — сброс прогрессии
Алгоритм:
День 1: 5
День 2: 7
День 3: 10
День 4: 15
День 5: 20
День 6: 25
День 7+: 30
При пропуске: сброс до 5
Сценарий:
Пользователь → /daily
Бот → проверяет время последнего бонуса
Если < 24 часов → "Вы уже забирали бонус сегодня. Следующий бонус через X часов"
Если > 24 часов → начисляет бонус и обновляет streak (или сбрасывает при пропуске)

2.6. Магазин товаров
Структура товара в БД
CREATE TABLE products (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    description TEXT,
    price INTEGER NOT NULL,
    delivery_type VARCHAR(20) DEFAULT 'auto',  -- 'auto' или 'manual'
    delivery_content TEXT,                     -- текст/ссылка/промокод (для auto)
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT NOW()
);
Типы выдачи товара
auto — бот присылает текст/ссылку/промокод автоматически
manual — администратор получает уведомление и выдаёт товар вручную
Сценарий покупки (auto)
Пользователь → /shop
Бот → показывает список активных товаров:

🛒 МАГАЗИН

1. 🔮 Доступ в VIP-чат — 500 кристаллов
   Закрытый чат с топ-разработчиками

2. 🔮 Промокод на 1000 руб. — 2000 кристаллов
   В любой онлайн-школы

3. 🔮 Консультация с экспертом — 1500 кристаллов
   30 минут личного общения

Выберите товар, нажав на кнопку.

Кнопки:
[1] [2] [3]
Пользователь → нажимает [1]
Бот → проверяет баланс
Если баланс >= 500:
  - списывает 500 кристаллов
  - логирует: "Покупка: Доступ в VIP-чат"
  - присылает товар и новый баланс
Иначе:
  - сообщает о нехватке кристаллов
  - предлагает пополнить баланс
Сценарий покупки (manual)
Пользователь → покупает товар с delivery_type = 'manual'
Бот:
  - списывает кристаллы
  - создаёт запись в pending_orders
  - уведомляет пользователя: "Заказ оформлен. Администратор свяжется с вами."
  - уведомляет администраторов:

  📦 НОВЫЙ ЗАКАЗ №124
  Пользователь: @alex_dev (ID: 123456789)
  Товар: Консультация с экспертом
  Цена: 1500 кристаллов
  Время: 15.04.2026 14:30
Таблица pending_orders:
CREATE TABLE pending_orders (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(telegram_id),
    product_id INTEGER REFERENCES products(id),
    status VARCHAR(20) DEFAULT 'pending',  -- pending, completed, cancelled
    ordered_at TIMESTAMP DEFAULT NOW(),
    completed_at TIMESTAMP,
    admin_comment TEXT
);

2.7. Платёжная система (ЮKassa)
Режимы
Тестовый: оплаты нет, кристаллы зачисляются мгновенно (демо)
Боевой: интеграция с ЮKassa API
Сценарий пополнения
Пользователь → /topup или кнопка "💰 Пополнить баланс"
Бот → показывает варианты:

[100 кристаллов — 50 руб.]
[250 кристаллов — 100 руб.]
[500 кристаллов — 180 руб.]
[1000 кристаллов — 330 руб.]

Или ввод своей суммы (50–10 000 руб.)
Пользователь → выбирает пакет
Бот → создаёт платёж в ЮKassa
Бот → отдаёт payment_url (валиден 1 час)
ЮKassa → webhook
Бот → проверяет статус succeeded
Бот → начисляет кристаллы
Бот → логирует транзакцию
Бот → подтверждение пользователю
