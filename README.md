# ProfBot

Telegram-бот с внутренней экономикой, реферальной системой, магазином цифровых товаров и административной панелью.

## Возможности

### Пользовательский контур
- Регистрация через `/start`.
- Профиль пользователя (`/profile`) и история транзакций.
- Магазин (`/shop`) с покупкой товаров:
  - `AUTO` — мгновенная выдача контента.
  - `MANUAL` — создание заказа для обработки администратором.
- Реферальная система и бонусная механика.
- Статистика и рейтинг пользователей.
- Пополнение баланса через Telegram Payments.

### Админский контур
- Админ-меню (`/admin`).
- Управление пользователями (просмотр, начисление баланса, транзакции).
- Управление товарами (добавление/редактирование).
- Управление заказами (`/orders`) с закрытием manual-заказов.
- Общая статистика проекта (`/admin_stats`).

## Технологии
- Python 3.12
- aiogram 3
- SQLAlchemy (async)
- PostgreSQL
- Redis
- Alembic
- Docker / Docker Compose

## Структура проекта
- `bot/routers` — Telegram-хендлеры.
- `bot/services` — бизнес-логика.
- `bot/repositories` — доступ к данным.
- `bot/models` — ORM-модели.
- `bot/core` — конфигурация, БД, Redis, логирование.
- `tests` — unit-тесты.

## Быстрый запуск (Docker)
1. Создайте `.env` (см. раздел ниже).
2. Запустите:
   ```bash
   docker compose up --build
   ```
3. Бот поднимется вместе с PostgreSQL и Redis.

## Локальный запуск (без Docker)
1. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
2. Поднимите PostgreSQL и Redis.
3. Примените миграции:
   ```bash
   alembic upgrade head
   ```
4. Запустите бота:
   ```bash
   python main.py
   ```

## Переменные окружения
Минимально необходимые:
- `BOT_TOKEN` — токен Telegram-бота.
- `DATABASE_URL` — URL подключения к PostgreSQL.
- `PAYMENT_TOKEN` — payment provider token.
- `ADMINS` — список Telegram ID админов (через запятую).

Дополнительно:
- `REDIS_URL` или параметры подключения к Redis.
